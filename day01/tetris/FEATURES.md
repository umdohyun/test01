# Tetris 풀스택 애플리케이션 - 구현 기능

## ✅ 완료된 기능

### 1. 백엔드 (FastAPI)

#### 인증 시스템
- ✅ 이메일 기반 회원가입
  - 이메일 유효성 검사 (pydantic EmailStr)
  - 비밀번호 해싱 (SHA-256)
  - 중복 이메일 체크
- ✅ 로그인
  - 이메일/비밀번호 인증
  - Bearer Token 발급
- ✅ 로그아웃
  - 세션 토큰 무효화
- ✅ 토큰 기반 인증
  - HTTPBearer 보안 스키마
  - 모든 게임 API에 인증 적용

#### 데이터베이스 (SQLite)
- ✅ users 테이블
  - 사용자 정보 저장
  - 이메일 UNIQUE 제약조건
- ✅ sessions 테이블
  - 세션 토큰 관리
  - 사용자 연결 (Foreign Key)
- ✅ game_records 테이블
  - 게임 기록 저장
  - 점수, 레벨, 라인 수, 플레이 시간

#### 게임 기록 API
- ✅ POST /api/game-records
  - 게임 종료 시 자동 저장
  - 인증된 사용자만 접근
- ✅ GET /api/leaderboard
  - 상위 N명 조회 (기본 10명, 최대 20명)
  - 점수 내림차순 정렬
- ✅ GET /api/top-score
  - 전체 최고 점수 1개 조회
  - 게임 화면에 실시간 표시
- ✅ GET /api/my-stats
  - 사용자별 통계 (총 게임 수, 최고 점수, 평균 점수 등)
- ✅ GET /api/my-history
  - 개인 플레이 이력 조회

#### CORS 설정
- ✅ 모든 오리진 허용 (개발 환경)
- ✅ 모든 HTTP 메서드 허용

### 2. 프론트엔드 (Vanilla JavaScript)

#### 인증 UI (auth.html)
- ✅ 로그인 폼
  - 이메일/비밀번호 입력
  - Enter 키로 제출
- ✅ 회원가입 폼
  - 비밀번호 확인 필드
  - 클라이언트 측 유효성 검사
- ✅ 폼 전환 (로그인 ↔ 회원가입)
- ✅ 에러/성공 메시지 표시
- ✅ 자동 로그인 유지
  - localStorage에 토큰 저장
  - 이미 로그인된 경우 게임 페이지로 자동 리다이렉트

#### 게임 화면 (game.html/game.js)
- ✅ 사용자 정보 표시
  - 상단에 이메일 표시
  - 로그아웃 버튼
- ✅ 최고 점수 표시
  - 실시간 업데이트
  - 점수와 플레이어 이메일
- ✅ 로그인 체크
  - 미로그인 시 auth.html로 리다이렉트
- ✅ 게임 종료 시 자동 저장
  - 백엔드 API 호출
  - 비동기 처리

#### 리더보드 (leaderboard.html)
- ✅ 상위 20명 표시
- ✅ 순위별 차별화
  - 1위: 금색
  - 2위: 은색
  - 3위: 동색
- ✅ 상세 정보 표시
  - 이메일, 점수, 레벨, 라인 수, 플레이 날짜
- ✅ 자동 날짜 포맷팅
- ✅ 새로고침 버튼
- ✅ 게임하기 버튼

#### 랜딩 페이지 (index.html)
- ✅ 게임 소개
- ✅ 조작법 설명
- ✅ 점수 계산표
- ✅ 리더보드 링크 추가

### 3. 게임 로직

#### 기본 테트리스 기능
- ✅ 7종 테트로미노 (I, O, T, S, Z, J, L)
- ✅ 블록 회전 (Wall Kick)
- ✅ 좌우 이동, 빠른 낙하, 즉시 낙하
- ✅ 라인 클리어
- ✅ 점수 계산 (1~4줄, 레벨별 배율)

#### 고급 기능
- ✅ 고스트 블록 (예상 위치 표시)
- ✅ 다음 블록 미리보기
- ✅ 레벨 시스템
  - 시간 기반 (30초마다)
  - 라인 기반 (10줄마다)
- ✅ 레벨업 애니메이션
- ✅ 일시정지 (P키)

## 🎯 핵심 요구사항 달성

1. ✅ **FastAPI 백엔드 구축**
   - RESTful API 설계
   - SQLite 데이터베이스 연동

2. ✅ **이메일 회원가입/로그인**
   - 이메일 유효성 검사
   - 로그인 후에만 게임 플레이 가능

3. ✅ **게임 기록 저장**
   - 플레이할 때마다 DB에 기록
   - 점수, 레벨, 라인 수 저장

4. ✅ **최고 점수 표시**
   - 전체 사용자 중 최고 점수
   - 게임 화면에 실시간 표시
   - 플레이어 이메일 함께 표시

## 📊 데이터 흐름

```
사용자 → 회원가입/로그인 → 토큰 발급 → localStorage 저장
   ↓
게임 플레이
   ↓
게임 종료 → 기록 저장 (토큰 인증) → DB 저장
   ↓
최고 점수 갱신 확인 → 게임 화면 업데이트
   ↓
리더보드 조회 → 상위 기록 표시
```
