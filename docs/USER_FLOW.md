## 3. 시나리오별 플로우

### 3.1 시나리오 1: 새로운 프로젝트 시작

**목표**: 프로젝트 작업 목록을 칸반보드에 정리

```
1. [사용자] 브라우저에서 index.html 열기
   ↓
2. [시스템] 샘플 카드 4개 표시
   ↓
3. [사용자] 샘플 카드 모두 삭제
   ↓
4. [사용자] "요구사항 분석" 입력, "할 일" 선택, 추가
   ↓
5. [사용자] "UI 디자인" 입력, "할 일" 선택, 추가
   ↓
6. [사용자] "백엔드 개발" 입력, "할 일" 선택, 추가
   ↓
7. [사용자] "테스트 작성" 입력, "할 일" 선택, 추가
   ↓
8. [결과] "할 일" 컬럼에 4개 카드 표시
```

### 3.2 시나리오 2: 작업 진행 상황 업데이트

**목표**: 진행 중인 작업을 칸반보드에 반영

```
1. [사용자] "요구사항 분석" 카드를 "진행중" 컬럼으로 드래그
   ↓
2. [시스템] 카드 이동, 개수 업데이트
   ↓
3. [사용자] 2시간 후 작업 완료
   ↓
4. [사용자] "요구사항 분석" 카드를 "완료" 컬럼으로 드래그
   ↓
5. [시스템] 카드 이동, 개수 업데이트
   ↓
6. [사용자] "UI 디자인" 카드를 "진행중" 컬럼으로 드래그
   ↓
7. [결과] 
   - 할 일: 2장
   - 진행중: 1장
   - 완료: 1장
```

### 3.3 시나리오 3: 하루 종료 및 다음 날 재개

**목표**: 작업 상태 저장 및 복원

```
[Day 1 - 오후 6시]
1. [사용자] 작업 중인 카드들의 상태 확인
   - 할 일: 3장
   - 진행중: 1장
   - 완료: 2장
   ↓
2. [사용자] 브라우저 닫기
   ↓
3. [시스템] localStorage에 자동 저장됨

[Day 2 - 오전 9시]
4. [사용자] 브라우저에서 다시 열기
   ↓
5. [시스템] localStorage에서 데이터 로드
   ↓
6. [시스템] 이전 상태 그대로 복원
   - 할 일: 3장
   - 진행중: 1장
   - 완료: 2장
   ↓
7. [사용자] 이어서 작업 진행
```

```
┌────────────────────────────────┐

├────────────────────────────────┤
│          🎯 칸반 보드          │
├──────────┬──────────┬──────────┤
│  📝 할 일│ ⚙️ 진행중│ ✅ 완료  │
│   (2)   │   (1)    │   (1)    │
```
