# 칸반보드 - 제품 요구사항 정의서 (PRD)
## Product Requirements Document

---

## 1. 제품 개요

### 1.1 제품명
**칸반 보드 (Kanban Board)**

### 1.2 제품 비전
개인 및 소규모 팀을 위한 직관적이고 가벼운 작업 관리 도구를 제공하여, 프로젝트 진행 상황을 시각적으로 파악하고 효율적으로 작업을 관리할 수 있도록 지원한다.

### 1.3 목표
- 설치 없이 브라우저에서 즉시 사용 가능한 칸반보드 제공
- 직관적인 드래그 앤 드롭 인터페이스로 낮은 학습 곡선
- 데이터 영구 저장으로 사용성 향상
- 모든 디바이스에서 접근 가능한 반응형 디자인

### 1.4 타겟 사용자
- **주 사용자**: 개인 개발자, 프리랜서, 소규모 프로젝트 팀
- **부 사용자**: 학생, 작은 업무를 관리하는 직장인
- **사용 환경**: 데스크톱 브라우저 (Chrome, Firefox, Safari, Edge)

## 2. 제품 범위

### 2.1 포함 기능 (In Scope)
1. **사용자 인증** ✨ NEW
   - 이메일/비밀번호 회원가입 (Supabase Auth)
   - 이메일/비밀번호 로그인
   - GitHub OAuth 소셜 로그인
   - 로그아웃
   - 세션 관리 및 자동 갱신

2. **카드 관리**
   - 카드 생성 (제목 입력)
   - 카드 삭제
   - 카드 상태 변경 (드래그 앤 드롭)

3. **칸반 보드**
   - 3개 고정 컬럼: 할 일, 진행중, 완료
   - 컬럼별 카드 개수 표시
   - 빈 컬럼 상태 표시

4. **데이터 영속성**
   - localStorage 기반 자동 저장
   - 페이지 새로고침 후 데이터 유지

5. **사용자 인터페이스**
   - 반응형 디자인 (데스크톱/태블릿/모바일)
   - 드래그 중 시각적 피드백
   - 직관적인 아이콘과 색상
   - 사용자 정보 표시 (헤더)

### 2.2 제외 기능 (Out of Scope - v1.1)
- 다중 보드 관리
- 팀 협업 기능 (공유, 댓글)

- 알림 기능

- 이메일 인증 (선택적)

### 2.3 향후 개선 사항 (Future Roadmap)

- v1.3: 컬럼 커스터마이징 (추가/삭제/이름 변경)

## 3. 기능 요구사항

### 3.1 필수 기능 (Must Have)

#### F-001: 카드 생성
**우선순위**: 높음  
**설명**: 사용자가 새로운 작업 카드를 생성할 수 있다.

**요구사항**:
- 카드 제목 입력 필드 제공
- 초기 상태 선택 드롭다운 (할 일/진행중/완료)
- "추가" 버튼 또는 Enter 키로 카드 생성
- 빈 제목 입력 시 경고 메시지
- 제목 최대 길이: 100자

**성공 기준**:
- 카드가 선택한 컬럼에 즉시 표시됨
- 입력 필드가 자동으로 초기화됨
- 카드 개수가 업데이트됨

#### F-002: 카드 삭제
**우선순위**: 높음  
**설명**: 사용자가 불필요한 카드를 삭제할 수 있다.

**요구사항**:
- 각 카드에 삭제 버튼 (× 아이콘) 표시
- 삭제 전 확인 다이얼로그 표시
- 삭제 시 즉시 화면에서 제거
- 카드 개수 업데이트

**성공 기준**:
- 확인 후 카드가 완전히 제거됨
- localStorage에서도 삭제됨
- 컬럼 카드 개수가 올바르게 갱신됨

#### F-003: 카드 상태 변경 (드래그 앤 드롭)
**우선순위**: 높음  
**설명**: 사용자가 카드를 드래그하여 다른 컬럼으로 이동시킬 수 있다.

**요구사항**:
- 카드 클릭 시 `cursor: grab` 표시
- 드래그 시작 시 카드 반투명 처리
- 드롭 영역 하이라이트 표시
- 드롭 시 카드 상태 자동 업데이트
- 모든 브라우저에서 작동

**성공 기준**:
- 드래그 앤 드롭이 매끄럽게 작동
- 카드가 올바른 컬럼에 이동
- 상태가 localStorage에 저장됨
- 시각적 피드백이 명확함

#### F-004: 데이터 영속성
**우선순위**: 높음  
**설명**: 사용자의 카드 데이터가 브라우저에 저장되어 유지된다.

**요구사항**:
- localStorage를 사용한 자동 저장
- 페이지 로드 시 자동 복원
- 카드 추가/삭제/이동 시 즉시 저장
- 저장 실패 시 에러 로깅

**성공 기준**:
- 페이지 새로고침 후 모든 카드 유지
- 브라우저 재시작 후에도 데이터 유지
- 최초 방문 시 샘플 카드 표시

#### F-005: 반응형 디자인
**우선순위**: 높음  
**설명**: 모든 디바이스에서 최적화된 UI를 제공한다.

**요구사항**:
- 데스크톱 (1920px): 3컬럼 가로 배치
- 태블릿 (768px-992px): 3컬럼 세로 배치
- 모바일 (< 768px): 풀 너비 세로 배치
- 터치 타겟 최소 크기: 44x44px

**성공 기준**:
- 모든 화면 크기에서 UI가 깨지지 않음
- 가독성 유지
- 터치 디바이스에서 버튼 클릭 가능

### 3.2 선택 기능 (Nice to Have)

#### F-006: 카드 개수 표시
**우선순위**: 중간  
**설명**: 각 컬럼의 카드 개수를 실시간으로 표시한다.

#### F-007: 빈 상태 표시
**우선순위**: 낮음  
**설명**: 컬럼이 비어있을 때 안내 메시지를 표시한다.

#### F-008: 초기 샘플 데이터
**우선순위**: 중간  
**설명**: 최초 방문 사용자에게 4개의 샘플 카드를 제공한다.

## 4. 비기능 요구사항

### 4.1 성능
- **페이지 로드 시간**: 1초 이내
- **드래그 앤 드롭 반응 속도**: 60fps 유지
- **카드 렌더링 시간**: 100개 카드 기준 500ms 이내

### 4.2 사용성
- **학습 시간**: 신규 사용자가 5분 이내에 모든 기능 파악
- **에러 방지**: 실수로 인한 데이터 손실 최소화 (삭제 확인)
- **피드백**: 모든 액션에 대한 즉각적인 시각적 피드백

### 4.3 호환성
- **브라우저**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **해상도**: 375px ~ 2560px
- **운영체제**: Windows, macOS, Linux, iOS, Android

### 4.4 보안
- **XSS 방지**: 사용자 입력 HTML 이스케이프 처리
- **데이터 격리**: localStorage는 도메인별 격리
- **HTTPS**: 프로덕션 환경에서 HTTPS 사용 권장

### 4.5 접근성
- **키보드 내비게이션**: Tab, Enter, Escape 키 지원
- **색상 대비**: WCAG 2.1 AA 레벨 준수
- **화면 리더**: ARIA 레이블 제공 (향후)

### 4.6 유지보수성
- **코드 품질**: 명확한 함수명, 주석
- **모듈화**: HTML/CSS/JS 파일 분리
- **에러 처리**: try-catch 블록으로 에러 로깅

## 5. 사용자 스토리

### US-001: 카드 생성
**As a** 사용자  
**I want to** 새로운 작업 카드를 생성하고 싶다  
**So that** 내가 해야 할 일을 기록할 수 있다

**Acceptance Criteria**:
- Given 입력 필드에 "API 개발"을 입력하고
- When "추가" 버튼을 클릭하면
- Then "할 일" 컬럼에 "API 개발" 카드가 추가된다

### US-002: 카드 이동
**As a** 사용자  
**I want to** 카드를 다른 컬럼으로 드래그하고 싶다  
**So that** 작업 진행 상황을 업데이트할 수 있다

**Acceptance Criteria**:
- Given "할 일" 컬럼에 "API 개발" 카드가 있고
- When 해당 카드를 "진행중" 컬럼으로 드래그하면
- Then 카드가 "진행중" 컬럼에 나타난다

### US-003: 카드 삭제
**As a** 사용자  
**I want to** 완료된 카드를 삭제하고 싶다  
**So that** 보드를 깔끔하게 유지할 수 있다

**Acceptance Criteria**:
- Given "완료" 컬럼에 카드가 있고
- When 삭제 버튼을 클릭하고 확인하면
- Then 카드가 보드에서 사라진다

### US-004: 데이터 유지
**As a** 사용자  
**I want to** 페이지를 새로고침해도 데이터가 유지되길 원한다  
**So that** 매번 처음부터 다시 시작하지 않아도 된다

**Acceptance Criteria**:
- Given 카드 3개를 추가하고
- When 페이지를 새로고침하면
- Then 추가했던 카드 3개가 그대로 보인다

## 6. 제약사항

### 6.1 기술적 제약
- 외부 라이브러리/프레임워크 사용 불가 (순수 바닐라 JS)
- 백엔드 서버 없음 (클라이언트 전용)
- localStorage 용량 제한: 5MB
- HTML5 Drag and Drop API 의존성

### 6.2 비즈니스 제약
- 예산: 0원 (무료 오픈소스)
- 일정: 1일 내 MVP 구현
- 인력: 1인 개발

### 6.3 환경적 제약
- 인터넷 연결 불필요 (오프라인 작동)
- 설치 없이 브라우저에서 즉시 실행
- 모바일 터치 이벤트 제한적 지원

## 7. 성공 지표 (KPI)

### 7.1 사용성 지표
- **작업 완료 시간**: 카드 생성 → 이동 → 삭제 전체 프로세스 30초 이내
- **에러 발생률**: 사용자 액션 1000회당 에러 1회 미만

### 7.2 성능 지표
- **초기 로딩 시간**: 1초 이내
- **드래그 앤 드롭 지연**: 50ms 이내
- **메모리 사용량**: 50MB 이내

### 7.3 품질 지표
- **브라우저 호환성**: 주요 브라우저 4종 100% 작동
- **반응형 대응**: 3가지 화면 크기 완벽 지원
